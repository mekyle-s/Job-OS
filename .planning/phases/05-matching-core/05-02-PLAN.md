**2. Create `src/lib/matching/ranker.ts` (per CONTEXT.md locked decision #5):**

Weighted ranking using 70% fit + 30% freshness. NO quality component (per CONTEXT.md locked decision #5, research suggested 60/30/10 but user locked 70/30).

Functions:
- `calculateFreshnessScore(daysOld: number): number` — Exponential decay: `e^(-0.1 * daysOld)`. Half-life ~7 days.
- `getRankedJobs(userId: string): Promise<RankedJob[]>` — SQL aggregation query that:
  1. Gets active, parsed jobs that match user's criteria (target companies)
  2. Counts total requirements and evidence-mapped requirements per job
  3. Calculates fit_score = mapped_requirements / total_requirements
  4. Calculates freshness_score using exponential decay on days since sourceUpdatedAt
  5. Computes composite_score = 0.7 * fit_score + 0.3 * freshness_score
  6. Determines fit band (High/Medium/Low per CONTEXT.md locked decision #4) using determineFitBand helper
  7. Generates top 2-3 reasons per job (from requirement categories with highest coverage, critical gaps)
  8. Returns ordered by composite_score DESC, limit 50

Per CONTEXT.md locked decision #4: Return fitBand (High/Medium/Low) with reasons. Keep internal numeric compositeScore for sorting but do NOT expose percentages to users. The reasons should reference specific evidence strength and critical gaps.

**5. Create `src/lib/matching/pipeline.ts`:**

Orchestrates the full matching pipeline for a single job.

Function: `runMatchingForJob(jobId: string, userId: string): Promise<MatchingResult>`

Pipeline steps:
1. Get job with requirements (verify job exists and has requirements)
2. Get user's evidence items with embeddings (skip items without embeddings)
3. For each requirement that doesn't already have a manual override mapping:
   a. If requirement has no embedding, generate one and store it
   b. Use findSimilarEvidence to get top-10 candidates
   c. For each candidate, call validateEvidenceMatch (LLM validation)
   d. Filter: keep match and weak_match, discard no_match
   e. Store valid mappings via createManyEvidenceMappings
4. Update job.lastMatchedAt timestamp
5. Return summary: { totalRequirements, mapped, gaps, needsReview }

Important per CONTEXT.md locked decision #7:
- DO NOT overwrite manual mappings (where createdBySystem=false)
- Set needsReview per decision mapping rules
- Include version info from versions.ts in every stored mapping

Per CONTEXT.md locked decision #9: Do not make performance claims. Log query times for monitoring but don't promise specific performance.

Hard limits per research Pitfall 4:
- Max 10 candidates per requirement (vector search top-K)
- Max 500 LLM calls per matching run (circuit breaker)
