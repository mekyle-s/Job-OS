**2. Create `src/app/providers.tsx`:**

Client component ('use client') that wraps children with QueryClientProvider.

```typescript
'use client';

import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { useState } from 'react';

export function Providers({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 1000 * 60 * 5, // 5 minutes
        refetchOnWindowFocus: true,
      },
    },
  }));

  return (
    <QueryClientProvider client={queryClient}>
      {children}
    </QueryClientProvider>
  );
}
```

**5. Create `src/lib/hooks/use-match-queue.ts`:**

Client hook for fetching the ranked match queue.

```typescript
'use client';

import { useQuery } from '@tanstack/react-query';
import { matchKeys } from './query-keys';

export function useMatchQueue() {
  return useQuery({
    queryKey: matchKeys.queue(),
    queryFn: async () => {
      const res = await fetch('/api/matching/queue');
      if (!res.ok) throw new Error('Failed to fetch match queue');
      return res.json();
    },
    staleTime: 1000 * 60 * 5, // 5 minutes
  });
}
```

**6. Create `src/lib/hooks/use-role-brief.ts`:**

Client hooks for role brief data and mapping mutations.

Functions:
- `useRoleBrief(jobId: string)` — Fetches role brief from /api/matching/[jobId]/brief
- `useRunMatching()` — Mutation that POSTs to /api/matching/run, invalidates queue and brief on success
- `useUpdateMapping(jobId: string)` — Mutation that PATCHes a mapping, invalidates brief and queue on success
- `useDeleteMapping(jobId: string)` — Mutation that DELETEs a mapping, invalidates brief and queue on success
- `useCreateMapping(jobId: string)` — Mutation that POSTs a new manual mapping, invalidates brief and queue on success

Cache invalidation strategy (per research Pattern 4, avoiding Pitfall 3):
- Running matching: invalidate queue AND specific brief
- Editing a mapping: invalidate specific brief AND queue (coverage changed)
- Deleting a mapping: invalidate specific brief AND queue
- Use granular query keys, not broad invalidation

Per CONTEXT.md locked decision #7: After manual mapping edits, invalidate brief cache but do NOT trigger re-matching.
