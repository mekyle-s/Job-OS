<objective>
Build the Fresh Match Queue UI page and Role Brief UI page with proof mapping display, gap identification, and manual mapping controls.

Purpose: This is where the user interacts with the matching engine. The queue page shows ranked opportunities with fit scores and next actions. The role brief page shows the requirement-level proof mapping that is the core product value — exactly which evidence proves each requirement, what gaps exist, and what to emphasize.

Output: Queue page, role brief page, and three reusable matching UI components (role card, brief view, mapping editor) using TanStack Query for data fetching and cache invalidation.
</objective>
