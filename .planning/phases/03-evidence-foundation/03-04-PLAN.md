**Create src/components/evidence/resume-upload.tsx** (Client Component, 'use client'):

File upload component with:

1. **Drag-and-drop zone**: Dashed border area (border-2 border-dashed rounded-lg p-8 text-center). Changes style on drag over (border-blue-500 bg-blue-50).
2. **File input**: Hidden, triggered by click on drop zone. Accept attribute: `.pdf,.docx,application/pdf,application/vnd.openxmlformats-officedocument.wordprocessingml.document`
3. **File type validation**: Client-side check that file is PDF or DOCX. Show inline error if wrong type.
4. **File size validation**: Max 4MB. Show inline error if too large. Format size display (e.g., "2.3 MB").
5. **Upload state**: idle | uploading | complete | error
   - idle: Show drop zone with icon and text "Drop your resume here or click to browse" + "PDF or DOCX, max 4MB"
   - uploading: Show progress indicator (spinner + "Uploading..."), disable drop zone
   - complete: Show success message and pass sourceId to parent via onUploadComplete callback
   - error: Show error message with "Try again" button

6. **Upload logic**: On file selection/drop:
   - Create FormData with file
   - POST to /api/evidence/upload
   - On success (200): extract sourceId from response, call onUploadComplete(sourceId)
   - On error: show error message from response

Props: `{ onUploadComplete: (sourceId: string) => void }`

Use native fetch, no external upload libraries (keeping dependencies minimal for MVP).

**Create src/components/evidence/parse-status.tsx** (Client Component, 'use client'):

Polling component that shows parsing progress:

Props: `{ sourceId: string }`

1. Poll /api/evidence/status/{sourceId} every 2 seconds using useEffect + setInterval
2. Display status with visual indicators:
   - pending: Clock icon + "Queued for parsing..."
   - processing: Spinner + "Parsing your resume with AI..." + animated dots
   - completed: Checkmark icon + "Resume parsed successfully! {itemCount} evidence items found." + "View Evidence" link to /dashboard/evidence
   - failed: Error icon + "Parsing failed: {error}" + "Try uploading again" link back to upload

3. Stop polling when status is 'completed' or 'failed' (clear interval)
4. Auto-redirect to /dashboard/evidence after 3 seconds on 'completed' (use router.push)

Use simple Tailwind styling. Status changes should feel smooth (no layout jumps).

**Create src/app/dashboard/evidence/upload/page.tsx** (mixed Server/Client):

1. Wrap page content in a client component for state management
2. Auth check via requireUser() in server component wrapper OR handle auth at API level (upload API already checks auth)
3. Page layout:
   - Header: "Upload Resume" with back link to /dashboard/evidence
   - Breadcrumb: Dashboard > Evidence Bank > Upload Resume
   - When no sourceId: Show ResumeUpload component
   - When sourceId received (after upload): Show ParseStatus component
   - Info text: "Upload your resume and we'll extract your experiences, projects, skills, and education automatically."

Since this needs client-side state transitions (upload -> polling), create an inner client component (e.g., UploadFlow) that manages the sourceId state:

```
function UploadFlow() {
  const [sourceId, setSourceId] = useState<string | null>(null);

  if (sourceId) return <ParseStatus sourceId={sourceId} />;
  return <ResumeUpload onUploadComplete={setSourceId} />;
}
```

The page.tsx can be a Server Component that renders the UploadFlow client component.
