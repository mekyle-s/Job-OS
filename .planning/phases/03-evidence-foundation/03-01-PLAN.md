Add two new tables to src/lib/db/schema.ts (keep all existing tables intact):

**1. evidenceSource table** - tracks uploaded files and source metadata:

```
evidenceSource = pgTable('evidence_source', {
  id: text('id').primaryKey(),           // Generate with crypto.randomUUID() or nanoid
  userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  sourceType: text('source_type').notNull(),  // 'resume' | 'github' | 'portfolio' | 'manual'
  fileName: text('file_name'),           // Original file name for uploads
  fileUrl: text('file_url'),             // Storage URL (local path or blob URL)
  fileSize: integer('file_size'),        // File size in bytes
  mimeType: text('mime_type'),           // 'application/pdf' or DOCX mime type
  parseStatus: text('parse_status').notNull().default('pending'),  // 'pending' | 'processing' | 'completed' | 'failed'
  parseError: text('parse_error'),       // Error message if parsing failed
  rawText: text('raw_text'),             // Extracted plain text from file
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull(),
}, (table) => [
  index('evidence_source_user_id_idx').on(table.userId),
  index('evidence_source_parse_status_idx').on(table.parseStatus),
])
```

**2. evidenceItem table** - individual evidence entries parsed from sources or manually created:

```
evidenceItem = pgTable('evidence_item', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' }),
  sourceId: text('source_id').references(() => evidenceSource.id, { onDelete: 'set null' }),  // null for manual entries

  itemType: text('item_type').notNull(),     // 'experience' | 'project' | 'skill' | 'education'
  title: text('title').notNull(),            // Role title, project name, skill name, degree
  company: text('company'),                  // Company/org name
  startDate: text('start_date'),             // YYYY-MM format
  endDate: text('end_date'),                 // YYYY-MM format, null if current

  content: text('content'),                  // Main description text
  metadata: jsonb('metadata').$type<{
    skills?: string[];
    technologies?: string[];
    achievements?: string[];
    links?: { type: string; url: string }[];
    location?: string;
    gpa?: string;
  }>(),

  confidence: real('confidence').default(1.0),    // 0.0-1.0, LLM-reported or 1.0 for manual
  isManual: boolean('is_manual').default(false).notNull(),  // true if user-created, false if parsed

  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull(),
}, (table) => [
  index('evidence_item_user_id_idx').on(table.userId),
  index('evidence_item_source_id_idx').on(table.sourceId),
  index('evidence_item_item_type_idx').on(table.itemType),
])
```

Import needed types: `import { pgTable, text, timestamp, boolean, real, jsonb, integer, index } from 'drizzle-orm/pg-core';`

Note: Use singular table names for consistency with Better Auth convention (DEV-012). Do NOT add vector/embedding columns yet - defer to Phase 5 when actually needed.

**Create src/lib/schemas/evidence.ts** - Zod schemas for evidence validation:

1. **ResumeEvidence schemas** (used by OpenAI Structured Outputs in Plan 03-02):
   - `ExperienceSchema`: role (string), company (string), startDate (string, YYYY-MM), endDate (string nullable), location (string optional), achievements (string[]), skills (string[]), confidence (number 0-1)
   - `ProjectSchema`: name (string), description (string), technologies (string[]), url (string optional), achievements (string[]), confidence (number 0-1)
   - `EducationSchema`: degree (string), institution (string), graduationDate (string nullable), location (string optional), gpa (string optional)
   - `ResumeEvidenceSchema`: { experiences: ExperienceSchema[], projects: ProjectSchema[], skills: string[], education: EducationSchema[] }

2. **CRUD validation schemas** (used by API routes and forms):
   - `EvidenceItemCreateSchema`: { itemType (enum: experience|project|skill|education), title (string min 1), company (string optional), startDate (string optional), endDate (string optional), content (string optional), metadata (object optional), confidence (number 0-1 optional, default 1.0), isManual (boolean optional, default true) }
   - `EvidenceItemUpdateSchema`: same fields as create but all optional (partial update)

Export all schemas and their inferred TypeScript types using z.infer.

**Create src/lib/db/queries/evidence.ts** - typed CRUD functions:

Import db from '@/lib/db', schema tables from '@/lib/db/schema', and eq/and/desc from 'drizzle-orm'.

Use crypto.randomUUID() for generating IDs (consistent with Better Auth's text ID approach from DEV-008).

Functions to implement:

1. `createEvidenceSource(data: { userId, sourceType, fileName?, fileUrl?, fileSize?, mimeType? })` - inserts new source, returns it. Generate ID with crypto.randomUUID().

2. `updateEvidenceSourceStatus(id: string, status: string, error?: string, rawText?: string)` - updates parse_status (and optionally parse_error, raw_text), sets updatedAt to new Date().

3. `getEvidenceSourcesByUser(userId: string)` - returns all sources for user, ordered by createdAt desc.

4. `getEvidenceSourceById(id: string)` - returns single source or undefined.

5. `createEvidenceItem(data: { userId, sourceId?, itemType, title, company?, startDate?, endDate?, content?, metadata?, confidence?, isManual? })` - inserts new evidence item, returns it. Generate ID with crypto.randomUUID().

6. `createManyEvidenceItems(items: array)` - batch insert for parsed resume items. Use db.insert().values(items) with pre-generated IDs.

7. `getEvidenceItemsByUser(userId: string)` - returns all evidence items for user, ordered by createdAt desc.

8. `getEvidenceItemById(id: string, userId: string)` - returns single item matching both id AND userId (security: prevent accessing other users' items).

9. `updateEvidenceItem(id: string, userId: string, data: partial)` - updates evidence item, sets updatedAt. Returns updated item. Verify userId matches before update.

10. `deleteEvidenceItem(id: string, userId: string)` - deletes evidence item. Returns deleted item. Verify userId matches.

All functions should be async and properly typed with return types matching Drizzle's InferSelectModel.
