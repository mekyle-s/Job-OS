**Create `src/lib/jobs/sources/adapter.ts`:**

Define the base adapter interface that all job sources must implement:

```typescript
export interface UserCriteriaInput {
  userId: string;
  jobFunction?: string | null;
  locations?: string[] | null;
  visaRequired?: boolean | null;
  targetCompanies: string[];
}

export interface RawJobData {
  sourceId: string; // ID from the source API
  sourceName: string; // 'greenhouse' | 'lever'
  rawData: unknown; // Complete source response preserved
}

export interface CanonicalJob {
  sourceId: string;
  sourceName: string;
  title: string;
  company: string; // Company name (not board token)
  location: string;
  description: string; // Full HTML/markdown content
  url: string; // Application URL
  postedAt: string | null; // ISO timestamp
  updatedAt: string; // ISO timestamp from source
  metadata: {
    departmentName?: string;
    officeLocation?: string;
    employmentType?: string; // 'full-time' | 'part-time' | 'internship'
    remote?: boolean;
    [key: string]: unknown;
  };
}

export interface JobSource {
  name: string;

  /** Fetch jobs matching user criteria from this source */
  fetchJobs(criteria: UserCriteriaInput): Promise<RawJobData[]>;

  /** Normalize raw source job to canonical model */
  normalizeJob(rawJob: RawJobData): CanonicalJob;

  /** Get all active source job IDs for a company (for inactive detection) */
  getActiveJobIds(boardToken: string): Promise<string[]>;
}
```

**Create `src/lib/jobs/sources/greenhouse.ts`:**

Implement the Greenhouse adapter:

```typescript
export class GreenhouseAdapter implements JobSource {
  name = 'greenhouse';
  private baseUrl = 'https://boards-api.greenhouse.io/v1/boards';
```

Key implementation details:

1. **Board token mapping:** V1 uses a hardcoded mapping of company name to Greenhouse board token. Include 10-15 well-known tech companies that use Greenhouse (e.g., "airbnb", "stripe", "figma", "notion", "twitch", "cloudflare", "discord", "databricks", "plaid", "ramp"). The mapping should be `Record<string, string>` where keys are lowercase company names and values are board tokens.

   IMPORTANT: Board tokens are the URL slugs used on `boards.greenhouse.io/{token}`. Research common ones. Some examples:
   - Airbnb: "airbnb"
   - Stripe: "stripe"
   - Cloudflare: "cloudflare"
   - Discord: "discord"

2. **fetchJobs(criteria):** For each targetCompany in criteria:
   - Look up board token from mapping (skip if not found)
   - Fetch `GET ${baseUrl}/${boardToken}/jobs?content=true`
   - Handle 429 (rate limit) with simple retry after 2 seconds
   - Handle non-200 responses gracefully (log and continue to next company)
   - Filter results by criteria.locations (case-insensitive substring match on `job.location.name`)
   - Filter by criteria.jobFunction (case-insensitive substring match on department names)
   - Return array of RawJobData with rawData preserving complete API response
   - Process companies sequentially (not parallel) to avoid rate limits per Pitfall 1

3. **normalizeJob(rawJob):** Convert Greenhouse job shape to CanonicalJob:
   - `company`: Use the board token mapping in reverse, or extract from metadata/context. Store the human-readable company name, NOT the board token.
   - `location`: `job.location.name` or "Not specified"
   - `description`: `job.content` (HTML)
   - `url`: `job.absolute_url`
   - `postedAt`: `job.created_at` (ISO string from Greenhouse, may need parsing)
   - `updatedAt`: `job.updated_at`
   - `metadata.employmentType`: Detect from title — if title contains "intern" (case-insensitive) → "internship", "part-time" → "part-time", else "full-time"
   - `metadata.remote`: `job.location.name` contains "remote" (case-insensitive)
   - `metadata.departmentName`: `job.departments[0]?.name`
   - `metadata.officeLocation`: `job.offices[0]?.name`

4. **getActiveJobIds(boardToken):** Fetch all jobs (without content for speed) and return array of source job IDs as strings. This enables the poller to detect removed jobs.

**Create `src/lib/jobs/sources/index.ts`:**

Adapter registry:

```typescript
import { GreenhouseAdapter } from './greenhouse';
import type { JobSource } from './adapter';

const adapters: Record<string, JobSource> = {
  greenhouse: new GreenhouseAdapter(),
};

export function getAdapter(source: string): JobSource | null {
  return adapters[source] ?? null;
}

export function getAllAdapters(): JobSource[] {
  return Object.values(adapters);
}
```

   You are a conservative job requirement extractor. Extract ONLY explicitly stated requirements from job postings.

   CRITICAL RULES:
   1. NEVER infer or assume requirements not clearly stated
   2. If a requirement is ambiguous, mark priority as "unknown"
   3. Preserve verbatim source text exactly as written
   4. Categorize accurately: technical_skill, experience, education, soft_skill, other
   5. Detect priority from keywords:
      - "required", "must have", "essential", "minimum" → required
      - "preferred", "nice to have", "bonus", "plus", "ideally" → preferred
      - No clear signal → unknown

   CATEGORIES:
   - technical_skill: Programming languages, frameworks, tools, technologies, platforms
   - experience: Years of experience, previous roles, domain knowledge
   - education: Degrees, certifications, courses, academic requirements
   - soft_skill: Communication, teamwork, leadership, problem-solving, collaboration
   - other: Anything else (visa status, availability, travel, clearance, etc.)

   CONSERVATIVE EXTRACTION EXAMPLES:
   ✓ "3+ years of Python experience" → Extract (explicit)
   ✓ "Bachelor's degree in Computer Science" → Extract (explicit)
   ✗ "Experience building scalable systems" → Do NOT expand to "distributed systems"
   ✗ "Strong communicator" → Extract as-is, do NOT expand to "written and verbal communication"

   When in doubt, extract less rather than more. Fewer false positives is better than comprehensive coverage.
