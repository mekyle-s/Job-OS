**Create `src/lib/jobs/workers/job-poller.ts`:**

pg-boss worker that processes a single user's polling job. Follow the established worker pattern from `src/lib/jobs/workers/resume-parser.ts` (per DEV-015: workers accept `Job<T>[]` array signature, extract `jobs[0]`).

Worker payload type:

```typescript
interface PollJobsPayload {
  userId: string;
  criteriaId: string;
}
```

Worker handler flow:

1. Extract payload from `jobs[0].data` (per DEV-015 array pattern)
2. Get user criteria from database using `getUserCriteria` query
3. Validate criteria exists and is active
4. Instantiate adapter using `getAdapter('greenhouse')` from sources/index.ts
5. Call `adapter.fetchJobs(criteria)` to get raw jobs
6. For each raw job:
   a. Store raw source record via `upsertRawJobSource()`
   b. Normalize via `adapter.normalizeJob(rawJob)`
   c. Upsert canonical job via `upsertJob()` — returns `{ job, isNew, isUpdated }`
   d. If `isNew`, queue requirement extraction: `boss.send('extract-requirements', { jobId: job.id, description: job.description })`
7. For each company in criteria.targetCompanies:
   a. Get active job IDs from adapter: `adapter.getActiveJobIds(boardToken)`
   b. Mark missing jobs as inactive via `markJobsInactive()`
8. Update `lastPolledAt` on the criteria record
9. Return summary: `{ userId, jobsFetched, newJobs, updatedJobs }`

Error handling:

- If adapter.fetchJobs() fails for one company, log error and continue to next (don't fail entire poll)
- If upsertJob() fails for one job, log error and continue
- Only throw if the entire poll fails (no criteria found, adapter missing)

**Create `src/lib/jobs/workers/requirement-parser.ts`:**

pg-boss worker that extracts requirements from a single job description. Follow the same worker pattern.

Worker payload type:

```typescript
interface ExtractRequirementsPayload {
  jobId: string;
  description: string;
}
```

Worker handler flow:

1. Extract payload from `jobs[0].data`
2. Update job parse status to 'processing' via `updateJobParseStatus(jobId, 'processing')`
3. Call `extractRequirements(description)` from requirement-extractor.ts
4. Store extracted requirements via `createRequirements(jobId, extracted.requirements)`
5. Update job parse status to 'completed' via `updateJobParseStatus(jobId, 'completed')`
6. Return summary: `{ jobId, requirementsExtracted: requirements.length, extractionNotes }`

Error handling:

- On failure, update job parse status to 'failed' with error message
- Re-throw error so pg-boss marks the job as failed (enables retry)

IMPORTANT: Import PgBoss `Job` type for proper typing:

```typescript
import type { Job } from 'pg-boss';
```

Both workers should be exported as named handler functions (e.g., `jobPollerHandler`, `requirementParserHandler`) so they can be registered in the jobs index.

**Update `src/lib/jobs/index.ts`:**

Add a worker registration function that can be called from the cron/poll endpoints:

```typescript
// Track registered workers to ensure idempotency
const registeredWorkers = new Set<string>();

export async function registerJobWorkers(boss: PgBoss): Promise<void> {
  // Only register if not already registered (idempotent)
  if (!registeredWorkers.has('poll-jobs-for-user')) {
    await boss.work('poll-jobs-for-user', { teamSize: 2 }, jobPollerHandler);
    registeredWorkers.add('poll-jobs-for-user');
  }

  if (!registeredWorkers.has('extract-requirements')) {
    await boss.work('extract-requirements', { teamSize: 3 }, requirementParserHandler);
    registeredWorkers.add('extract-requirements');
  }
}
```

Import the worker handlers at the top:
