    - Export `type RoleStatus = z.infer<typeof RoleStatusEnum>`
    - Export `UpdateRoleStatusSchema = z.object({ status: RoleStatusEnum, notes: z.string().max(500).optional() })`
    - Export `RoleStatusFilterSchema = z.enum(['all', 'ignore', 'save', 'apply', 'applied'])` for queue filtering

    **2. Create src/lib/db/queries/role-status.ts:**
    - `upsertRoleStatus(userId: string, jobId: string, status: string, notes?: string)` - Upsert using onConflictDoUpdate on (userId, jobId) unique constraint. Generate id with crypto.randomUUID(). Set statusChangedAt and updatedAt on conflict update.
    - `getRoleStatusForJob(userId: string, jobId: string)` - Single status lookup
    - `getRoleStatusesForUser(userId: string, statusFilter?: string)` - All statuses for user, optional filter by status value. Returns array of {jobId, status, notes, statusChangedAt}.
    - `deleteRoleStatus(userId: string, jobId: string)` - Remove status

    **3. Create src/lib/db/queries/audit.ts:**
    - `logParserAudit(params)` accepting: userId (nullable for system), entityType, entityId, action, parserConfidence (optional), beforeValue (optional), afterValue (optional), source (default 'user'), userFeedback (optional)
    - Generate id with crypto.randomUUID()
    - Insert into parserAudit table
    - This is a fire-and-forget helper -- catch and log errors but don't throw (audit should not block business operations)
    - `getAuditTrail(entityType: string, entityId: string)` - Get audit entries for an entity, ordered by createdAt desc
