    ```typescript
    export const roleStatusKeys = {
      all: ['roleStatus'] as const,
      forJob: (jobId: string) => [...roleStatusKeys.all, jobId] as const,
      forUser: () => [...roleStatusKeys.all, 'user'] as const,
    };
    ```

    **3. Create src/lib/hooks/use-role-status.ts:**
    - `useRoleStatus(jobId: string)` - Query hook that fetches GET /api/roles/{jobId}/status, returns { status, notes }
    - `useUpdateRoleStatus()` - Mutation hook:
      - mutationFn: PATCH /api/roles/{jobId}/status with { status, notes? }
      - onMutate: Optimistic update -- cancel in-flight queue queries, snapshot previousQueue, update cache by setting job's status in the queue data
      - onError: Rollback to previousQueue snapshot
      - onSettled: Invalidate matchKeys.queue() AND roleStatusKeys.forJob(jobId)
    - Both hooks use 'use client' directive and import from '@tanstack/react-query'

    **4. Create src/app/dashboard/queue/filters.tsx:**
    - 'use client' component
    - Import useQueryState and parseAsStringLiteral from 'nuqs'
    - Define statusValues: ['all', 'ignore', 'save', 'apply', 'applied'] as const
    - Use useQueryState('status', parseAsStringLiteral(statusValues).withDefault('all'))
    - Render a horizontal flex row of filter buttons with labels: All, Ignore, Save, Apply, Applied
    - Active filter gets bold text + colored bottom border (like a tab bar)
    - Each button calls setStatus(value) on click
    - Export both the component and a useStatusFilter hook that returns current status value
    - Style: pill/tab buttons with bg-gray-100 default, bg-blue-100 text-blue-800 for active

    **5. Update src/app/dashboard/queue/page.tsx:**
    - Import QueueFilters from './filters'
    - Import NuqsAdapter from 'nuqs/adapters/next/app' (required for Next.js App Router)
    - Wrap the page content in <NuqsAdapter> (nuqs requires this provider in App Router)
    - Place <QueueFilters /> between the header and the queue list
    - Add client-side filtering: read ?status from URL params via useQueryState in the page OR pass filter to useMatchQueue
    - Filter the queue array client-side: if status !== 'all', filter where job's roleStatus matches the filter
    - NOTE: This requires the queue API to include role status per job. Two approaches:
      a. (Simpler) Fetch role statuses client-side alongside queue, merge in component
      b. (Better) Update GET /api/matching/queue to join with roleStatus and include status field
    - Choose approach (a) for now: use a separate useRoleStatusesForUser() hook OR fetch statuses in the page and merge with queue data
    - Actually, simplest approach: filter happens after queue loads. Each RoleCard fetches its own status via useRoleStatus(jobId). The page-level filter reads URL state and filters the rendered cards by their fetched status. For V1, this is acceptable with <50 cards.

    - Current status button gets highlighted/active styling (filled background)
    - Buttons are disabled while mutation isPending (prevent race condition per research Pitfall 5)
    - Stop click propagation on status buttons (the card is wrapped in a Link, so e.stopPropagation() + e.preventDefault() on status button clicks)
    - Show the current status as a small badge next to the fit band if status is set

    Styling notes:

    - Colors: ignore=gray-200/gray-700, save=blue-100/blue-700, apply=yellow-100/yellow-700, applied=green-100/green-700
