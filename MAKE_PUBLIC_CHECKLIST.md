# ✅ Make Public Checklist

Use this checklist before making your repo public!

## 🔒 Security Check

- [ ] Verify `.env.local` is in `.gitignore` ✅ (already done)
- [ ] Confirm no API keys in git history ✅ (already done)
- [ ] Ensure `uploads/` folder is gitignored ✅ (already done)
- [ ] Double-check no sensitive data in `.planning/` docs
- [ ] Remove any personal email addresses or phone numbers from code

## 📝 Documentation

- [ ] ✅ README.md created
- [ ] ✅ LICENSE added (MIT)
- [ ] ✅ CONTRIBUTING.md added
- [ ] Update contact info in README.md (see **Action Required** below)
- [ ] Review README for typos/clarity

## 📸 Screenshots (Optional but Recommended)

- [ ] Read `SCREENSHOTS_TODO.md`
- [ ] Capture 4-5 key screenshots (15-20 min sprint)
- [ ] Create `screenshots/` folder
- [ ] Add screenshot embeds to README.md
- [ ] Delete `SCREENSHOTS_TODO.md` after done

## 🎨 Polish (Nice to Have)

- [ ] Add a logo or banner image to README
- [ ] Create a demo video (Loom, 2-3 min walkthrough)
- [ ] Deploy to Vercel and add live demo link
- [ ] Add project to GitHub topics: `internship`, `nextjs`, `ai`, `job-search`, `portfolio`

## 🚀 Pre-Launch Actions

- [ ] Test the README install instructions on a fresh machine (optional)
- [ ] Run `npm run lint` and fix any issues
- [ ] Ensure `npm run build` completes successfully
- [ ] Commit all changes

## 📢 Going Public

1. [ ] Go to GitHub repo settings
2. [ ] Scroll to "Danger Zone"
3. [ ] Click "Change repository visibility"
4. [ ] Select "Public"
5. [ ] Confirm by typing the repository name
